# hardware/ — HardFuzz v1 PCB

The custom board that turns the validated dev-board prototype (Cmod A7 + NUCLEO in the
repo root) into a plug-and-play product. Design captured here; the KiCad project lands in
`hardfuzz_v1/` once the schematic is drawn from the spec.

## What's here

| File | What it is |
|---|---|
| [schematic_design.md](schematic_design.md) | the electrical design — block diagram, power tree, part selection, and net-by-net connections per block. **Start here.** |
| [bom.csv](bom.csv) | bill of materials with part numbers, packages, and 100-unit costs (~$40 total) |
| [kicad_parts.md](kicad_parts.md) | per-part KiCad symbol/footprint readiness — what's stock vs. what to grab/draw before capture |
| [hardfuzz_v1/](hardfuzz_v1/) | KiCad 10 project — captured **as code** via [`gen/`](hardfuzz_v1/gen/) (kigen). Power sheet generated + netlist-verified; see [gen/README.md](hardfuzz_v1/gen/README.md) |

Product-side rationale (attach modes, out-of-box experience) lives in
[../docs/HardFuzz_v1_Spec.md](../docs/HardFuzz_v1_Spec.md).

## Design approach

Reuse the **proven Cmod A7 core** (Artix-7 XC7A35T die + FT2232H USB-JTAG/UART + QSPI
config; FTG256 package for easier fan-out) so the RTL we already run on hardware drops on
unchanged (rebuilt against a new pinout), and add the
productization layer: level-shifted bus I/O with `VREF` sensing, an onboard SN65HVD230 CAN
transceiver, and keyed target connectors (Qwiic / CAN screw terminal / SPI interposer).

## Status

**KiCad capture complete (schematic + placed PCB) — captured as code.** Rather than
hand-drawing, the whole design is generated by [`hardfuzz_v1/gen/`](hardfuzz_v1/gen/) (kigen)
and validated with `kicad-cli`. All five blocks — power, CAN, level-shifter+connectors,
USB/FT2232H+flash, and the **FPGA (XC7A35T-CSG324)** — are assembled into a **hierarchical**
schematic (`hardfuzz_v1.kicad_sch` + 5 readable sub-sheets, 68 parts / 302 nets), with
cross-sheet nets verified (SPI↔FPGA↔level-shifter, JTAG↔FT2232, flash↔FPGA, VREF↔XADC,
CAN↔FPGA). The **kicad-happy skill** reviewed it — every subsystem detected at HIGH trust.
`kigen_pcb.py` then generates **`hardfuzz_v1.kicad_pcb`**: all 68 footprints placed with
pads on the right nets + board outline (loads in pcbnew; the 317 "unconnected" are the
unrouted nets). Open the project as **`hardfuzz_v1.kicad_pro`**. Left for the GUI:
**routing**, final placement, passive MPNs. See [hardfuzz_v1/gen/README.md](hardfuzz_v1/gen/README.md).

## Doing the KiCad capture

The generator approach (see [hardfuzz_v1/gen/README.md](hardfuzz_v1/gen/README.md)):

1. Define each sheet's parts + nets in a `gen/<sheet>_sheet.py`, using net names from
   `schematic_design.md` §3–§9 so the netlist matches the RTL `.xdc` intent.
2. Regenerate and validate: `python3 gen/<sheet>_sheet.py` → `kicad-cli sch erc/netlist`.
3. Open `hardfuzz_v1/` in KiCad to route/prettify, assign footprints, and lay out the
   4-layer PCB per §11.

KiCad's own files are the source of truth once opened/edited; regenerate the BOM from the
schematic at that point (this `bom.csv` is the design-time reference).
